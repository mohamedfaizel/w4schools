Angular - Introduction

What is Angular
1. Framework to build Client side applications.
2. Special for SPAs (Single Page Applications).
Why Angular
1. Modular Approach - Application we buit in Clear stucture.
2. Re-usable Code - Making use of Components.
3. Development Quiker and Easy - Validation, Routing and so on
3. Unit testing - possible
4. Google + Microsoft
Angular 2 itself, is built using TypeScript. TypeScript has great support of ECMAScript 6 Standard.
Why Angular - 2
1. Performance : Angular 2 is 5 times faster then Angular 1
2. Mobile Support : with Angular 2 we can build a single application that work across mobile and desktop devices.
3. Component Based Development : In Angular 2, Everything is component. Components are the building blocks of an angular application.
Advantage : Code Reuse and unit testing
4. More Language Choice
  • ECMAScript 5
  • ECMAScript 6 (also called ES 2015)
  • TypeScript (is the most popular language)
  • Dart
  • PureScript

New Project Creation Step By Step


  • New Project always creating with routing
    ng new projectname --routing
  • Existing Project check "app-routing.module.ts". if doesn't exist, Please create it and register with "app-module.ts".
    ng g m app-routing --flat=true -m=app.module
  • Create a module for each section (Group of Components) like "authentication", "pages", "Admin" etc.. with module.
    ng g m main/authentication --routing
    • authentication - new Folder Created
    • authentication-routing.module.ts - new Routing module created.
    • authentication.module.ts - new section module created.
    Note : main is a sub folder for src > app > main,
  • Create a "Component" for pages and register with "section-routing.module".
    Example for authentication module pages.
    ng g c main/authentication/login --spec=false -m=authentication-routing
    ng g c main/authentication/forgot-password --spec=false -m=authentication-routing
    ng g c main/authentication/reset-password --spec=false -m=authentication-routing
    ng g c main/authentication/mail-confirm --spec=false -m=authentication-routing
    ng g c main/authentication/lock --spec=false -m=authentication-routing
  • Create "Pasge-not-found" Component in authentication folder but register with "app-routing.module".
    Because its global routing access
    ng g c main/authentication/page-not-found --spec=false -m=app-routing
  • Note: "-m=authentication-routing" automatically import and declared the component. otherwise it will do manually.
    component import and declaration in "section-routing.module" is best code practise.
    otherwise it will import in both place. Routes for section-routing-module and declartion for section.module.

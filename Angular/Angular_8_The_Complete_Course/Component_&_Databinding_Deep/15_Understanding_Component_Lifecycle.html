Understanding Component Life-cycle Hooks

If a new component is created in angular or initiated. Example, it will initiate a new version of that component and add it into the DOM.
So once a new Component is initiated, Angular Goes through a couple of different phases in this creation process and it will actually give us a chance to hook into these phases and execute some code.
Note:
  • 1st time component initiated
    1. Constructor()
    2. ngOnchanges()
    3. ngOnInit()
    4. ngDoCheck()
  • 2nd Time Onchange Button Clicked
    • ngOnchanges()
    • ngDoCheck()
  • 3rd time component called Again (AddServerButton())
    1. Constructor()
    2. ngDoCheck()
    3. ngOnchanges()
    4. ngOnInit()
    5. ngDoCheck()
  • Orders for all hooks component initiated
    1. Constructor()
    2. ngOnchanges()
    3. ngOnInit()
    4. ngDoCheck()
    5. ngAfterContentInit()
    6. ngAfterContentChecked()
    7. ngAfterViewInit()
    8. ngAfterViewChecked()
    9. ngOnDestroy()
Note-2:

@ViewChild Element we cant access the value in ngOnInit()but ngAfterViewInit() here we can access the @viewChild element value.

@ContentChild Element we cant access the value in ngOnInit()but ngAfterContentInit() here we can access the @viewChild element value.


Life Cycle of hooks
  1. ngOnChanges Called after a bound input property changes
    • It executes multiple times
    • After the constructor() called, This is first hook called
    • It's execute right at the start when a new component is created
    • thereafter. it's also always called whenever one of our bound properties changes and with that, I mean Properties decorated @input, so whenever these properties received new values.
    • This is the only hook have the parameters (ngOnchanges(changes: SimpleChanges){})
  2. ngOnInit Called once the component is initialized
    • This is second hook
    • this method gets executed once the component has been initialized.
    • It has not been added to the DOM. it has not been displayed yet but angular finished the basic initialization, our properties can now be accessed and initialized
    • Example : you could say "the Object was created."
    • ngOnInit will run after the constructor
  3. ngDoCheck Called during every change detection run
    • This method will executed a lot because this will run whenever change detection runs
    • ngDoCheck is a hook executed on every check Angular makes.now important, on every check
    • Any Events : like click, onChange, Selected, properties changes, value changes, timerfiered, observable and resolved,
    • Angular works pretty great and doesn't cost a lot of performance
    • ngDoCheck is a great method to use if you want to do something on every change detection cycle. like maybe manually inform Angular about some changes it would not be able to detect otherwise, through that is very advanced use case.
  4. ngAfterContentInit Clled after content (ng-content) has been projected into view
    • Especially the part which will get added to our component through ng-content.
  5. ngAfterContentChecked
    • It's executed whenever change detection checked this content we're projecting into pur component
  6. ngAfterViewInitCalled every time the view (and child Views) has been checked
    • The view of our own component has been finised initializing.
  7. ngAfterViewCheckedcalled every time the view(and child views) have been checked
    • Once we are sure that either all changes which had to be done were displayed in the view or no changes were detected by angular.
  8. ngOnDestroyCcalled once the component is about to be destroyed
    • Example : if you placed ngIf on it and this gets then set to false and therefore it removes it from the DOM, ngOnDestroy is called
    • Here is the great placed to do some cleanup work because this is called right before the object itself will be destroy by Angular.

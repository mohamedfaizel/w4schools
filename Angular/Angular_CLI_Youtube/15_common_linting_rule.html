Linting Typescript

  • quotemark => string values rules specifies wheather you want "single" or "double" qoutes.
    Error => let message = "mesasge";
    Pass => let message = 'mesasge';
    Note: ng lint --fix => automatically fix this errors
  • no-trailing-whitespace => disallow trailing whitespace at the end of a line
    Error => let message = 'mesasge'; &nb_sp; => Error
    Pass => let message = 'mesasge';
    Note: ng lint --fix => automatically fix this errors
  • semicolon => rule specifies that a line should be terminated with a semicolon.
    Error => let message = 'mesasge'
    Pass => let message = 'mesasge';
    Note: ng lint --fix => automatically fix this errors
  • comment-formatsingle line comment must begin with space
    Error => //type your comment
    Pass => // type your comment
  • component-class-suffix => rule enforce that a component class shold end with the suffix Component
    Error => export class App {...}
    Pass => export class AppComponent {...}
  • use-life-cycle-interface => that you add the implement keyword for every lifecycle hook you use
import { Component, OnInit} from '@angular/core'; export class AppComponent { ngOnInit: void{ } } import { Component, OnInit} from '@angular/core'; export class AppComponent implements OnInit{ ngOnInit: void{ } }
